Narrative AI Framework

The Narrative AI Framework is a developer-centric SDK engineered for building robust, conversational multimodal applications. It delivers production-grade utilities mapping Language Models, Audio processing (Speech-to-Text & Text-to-Speech), Vector Search/Memory (RAG), Vision Language capabilities (VLM), Document Restoration/OCR, and Live WebRTC voice pipelines. It is highly optimized for bilingual Arabic-English code-switching and Egyptian colloquial dialects.

Quickstart Installation

Install the core framework and its relevant packages easily from PyPI:

pip install narrative-ai-framework

narrative_ai.llm.generate(prompt, system_prompt=None, temperature=0.7, max_tokens=None, provider=None)
[source]
Generates a text completion based on the given prompt sequence and system setup instructions. Supports automatic routing across OpenAI and Gemini.
Parameters
prompt str required
The primary input prompt string or message flow context to generate text for.
system_prompt str default=None
Overriding developer instructions to configure model agent behavior and personality constraints.
temperature float default=0.7
Adjusts response creative variety. Higher temperature yields more random, creative text. Range is 0.0 to 2.0.
max_tokens int default=None
Hard threshold limit bounding the total count of output tokens generated.
provider str default=None
Specifies a specific model provider path ('openai', 'gemini') to override the current environment config.
Returns
LLMResult
An object package returning generated text attributes:
• text (str): Result text sequence.
• provider (str): Actual provider matching execution context.
• model (str): System model signature used.
• input_tokens (int): Calculated input tokens count.
• output_tokens (int): Calculated output tokens count.
Examples
python 
import narrative_ai as nai

# 1. Define prompt query & target system guidelines in Egyptian Arabic dialect
prompt_query = "اكتب قصة قصيرة عن سر الذكريات والمذكرات الشخصية."
system_instruction = "أنت مساعد ودود وموجز وتتكلم بالعامية المصرية."

# 2. Call the generator (routes automatically to OpenAI or Gemini based on config)
result = await nai.llm.generate(
    prompt=prompt_query,
    system_prompt=system_instruction,
    temperature=0.8,                  # Higher values increase creativity/variety
    max_tokens=150                    # Limit output tokens to control latency and costs
)

# 3. Print the execution statistics and textual outputs
print(f"Active Provider: {result.provider}")
print(f"Active Model: {result.model}")
print(f"Result Output:\n{result.text}")

narrative_ai.stt.speech_to_text(audio, language=None, extract_emotion=True, use_vad=True)
[source]
Transcribes raw audio bytes or files into written text formats. Specializes in handling English-Arabic code-switching scenarios seamlessly.
Parameters
audio bytes / str required
Either binary raw PCM audio waveform bytes, or a local system file path to an audio file (.wav, .mp3).
language str default=None
Manual code configuration ('ar', 'en'). Set None to automatically parse and determine active languages.
extract_emotion bool default=True
Enables acoustic profiling models to return primary emotion labels and confidence values.
use_vad bool default=True
Toggles Voice Activity Detection filters to discard ambient environmental noise.
Returns
TranscriptionResult
A transcription result object containing parsed properties:
• text (str): Transcribed dialogue string.
• confidence (float): Acoustic alignment match probability score.
• language (str): Classified language code.
• emotion (str): Acoustic emotion category ('happy', 'sad', 'angry', 'neutral').
• emotion_confidence (float): Confidence score of the detected emotion.
Examples
python 
import narrative_ai as nai

# 1. Load target diary audio recording bytes from local filesystem
with open("my_diary_entry.wav", "rb") as audio_file:
    raw_audio_bytes = audio_file.read()

result = await nai.stt.speech_to_text(
    audio=raw_audio_bytes,
    language="ar",             # Set language ('ar' or 'en', None for autodetection)
    extract_emotion=True,      # Toggles voice emotional analysis
    use_vad=True               # Use Voice Activity Detection to filter noise
)

# 3. Print transcription outcomes
print(f"Transcription Text: {result.text}")
print(f"Acoustic Emotion: {result.emotion} (Confidence: {result.emotion_confidence * 100:.1f}%)")
